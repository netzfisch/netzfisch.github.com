I'm looking for a solution to the "RVM is not a function" issue when installing RVM on Raspbian using LXTerminal. The following thread discusses workarounds for this issue:

rvm installation not working: "RVM is not a function"

One solution would be to start the terminal in "login mode" by doing /bin/bash --login, but this does not help on Raspbian because it uses LXTerminal. Other people have suggested putting the following line at the end of your .bash_rc file:

source ~/.rvm/scripts/rvm

...which seems to work. Just wondering if (a) the above workaround is legit and will not screw anything up and (b) if anybody has found a more elegant solution for RVM on Raspbian.

rvm debian raspberry-pi raspbian

It looks like the best solution is to run it with:

lxterminal -e "bash -li"
It will make the terminal use an login and interactive shell.

Deploying enki at heroku

Clone enki blog at github and then get a local copy:

pre(terminal). $ git clone git@github.com:netzfisch/enki.git
$ mv enki seitenmacher
$ cd seitenmacher
$ git checkout -b myblog

Adjust enki.yml, make sure not to use delegation, as yo need therefore 2 workers (heroku support request from/link):

pre(terminal). $ git commit -a -m 'customised enki.yml'
$ git add .gems
$ git commit -a -m 'adding gem manifest file for heroku'
$ git checkout master
$ git merge myblog
$ rake db:migrate

Prepare your local box for testing development versions:

pre(terminal). $ sudo apt-get install libxml2-dev libxslt1-dev
$ sudo gem install webrat --version 0.3
$ rake spec
$ script/server

Get ready for upload

pre(terminal). $ heroku create seitenmacher
$ git push heroku master
$ heroku rake db:migrate
$ heroku rake spec #check for errors, missing gems?

Installing Rails on Debian / Ubuntu

h1. Get Ruby

pre(bash). $ sudo apt-get update
$ sudo apt-get install build-essential libopenssl-ruby 
$ sudo apt-get install ruby ruby1.8-dev libsqlite3-dev

h2. Installing RubyGems

bq(code). $ sudo apt-get install rubygems

To make gem work properly add path to ~/.bashrc:

bq. export PATH=/var/lib/gems/1.8/bin:$PATH

h2. Installing Rails

Use RubyGems to install rails and dependencys:

bq. $ sudo gem install rails sqlite3-ruby

h2. Get missing gems

pre(terminal). $ $ gem list --local
sudo rake gem:install
